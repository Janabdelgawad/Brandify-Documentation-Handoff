**Authentication & Authorization**
- **Feature name:** User registration, email verification, login, logout, password reset, token-based auth  
- **Business purpose:** Onboard users, protect account actions, enable session management for dashboard/store owners.  
- **Technical implementation summary:** Express routes implement register/login/logout/forgot-password/reset/verify flows. Passwords hashed with `bcryptjs`. Email verification and password reset tokens generated with `crypto` and sent via `nodemailer`. JWTs issued and stored in HTTP-only cookie and `x-auth-token` header; `auth` middleware verifies JWT. Rate limiter guards login attempts.  
- **Main files involved:** auth.js, auth.js, User.js  
- **Routes/API endpoints involved:** `POST /api/auth/register`, `GET /api/auth/verify/:token`, `POST /api/auth/resend-verification`, `POST /api/auth/login`, `POST /api/auth/logout`, `POST /api/auth/forgotpassword`, `POST /api/auth/resetpassword/:token`, `GET /api/auth/user`  
- **Database models involved:** `User` (User.js)  
- **Frontend pages/components involved:** Register.jsx, Login.jsx, VerifyEmail.jsx, ForgotPassword.jsx, ResetPassword.jsx, auth usage in `AuthContext.jsx`  
- **Third-party services used:** `nodemailer` (SMTP via env), JWT, `bcryptjs`, `express-rate-limit`  
- **Current implementation status:** Implemented and wired to frontend; token expiry configured (1h) and server sets cookie.

**Security**

- **Business purpose:** Protect endpoints, limit abuse, validate access to premium features.  
- **Technical implementation summary:** `helmet()` applied app-wide; `cors` configured for frontend origin; various `express-rate-limit` uses (login, creative generation); `auth` JWT middleware across protected routes; `checkSubscription` middleware enforces `User.subscriptionStatus === "active"` and expiry handling. Upload input filtering via multer `fileFilter`.  
- **Main files involved:** server.js, auth.js, checkSubscription.js, route-specific rate limiters in creative.js and auth.js  
- **Routes/API endpoints involved:** All protected endpoints use `auth` (see routes list); subscription-sensitive features expected to use `checkSubscription` (middleware file exists).  
- **Database models involved:** `User`  
- **Frontend pages/components involved:** Auth-protected dashboard pages (various `dashboard/*` pages), API client uses `withCredentials=true` and adds `x-auth-token`.  
- **Third-party services used:** `helmet`, `express-rate-limit`, CORS, multer/cloudinary validations  
- **Current implementation status:** Implemented; basic protections present. Some sanitizers commented out (`express-mongo-sanitize`, `xss-clean`) in server.js.

**AI Creative Generator (Ad Creative Composer)**
- **Feature name:** Generate background scenes, remove product background, composite product + CTA + typography, deliver downloadable creatives  
- **Business purpose:** Produce ready-to-run ad creatives that preserve product identity and place it in premium AI-generated backgrounds with CTAs.  
- **Technical implementation summary:** Multi-step pipeline in [backend/routes/creative.js]:
  - Accepts product image (Multer memory storage).
  - Calls local Python enhancement (main.py) to remove background and produce transparent product (via `runEnhancement` and `rembg` pipeline).
  - Uploads the reference image to Cloudinary.
  - Calls image generation APIs (primary: Hugging Face FLUX model via HF API; fallback: Pollinations). Also analyzes product with Google Gemini Vision (via REST).  
  - Uses `sharp` to composite generated background + resized transparent product + SVG overlay (offer/CTA).
  - Upload final creative to Cloudinary and return URLs.
  - Rate-limited and uses exponential backoff for retry; parallel generation with Promise.allSettled.  
- **Main files involved:** creative.js, creative.js (module copy/variant), main.py, pipeline.py, AICreativeGenerator.jsx  
- **Routes/API endpoints involved:** `POST /api/creative/generate`, `GET /api/creative/platforms`  
- **Database models involved:** None directly (ad generation is transient); `User` auth required.  
- **Frontend pages/components involved:** AICreativeGenerator.jsx  
- **Third-party services used:** Hugging Face API (`HF_API_KEY`), Pollinations fallback, Cloudinary (uploads), Google Gemini Vision (`GEMINI_API_KEY`), `sharp`, Python `rembg` and OpenCV pipeline, `axios` for HTTP, Framer Motion on frontend.  
- **Current implementation status:** Fully implemented end-to-end from frontend to final Cloudinary upload; external API keys required. There is also a module copy under AI-Creative-Generator mirroring the route (duplicate/alternate entry).
